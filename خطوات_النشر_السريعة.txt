🚀 خطوات نشر التحديثات على Render - دليل سريع
═══════════════════════════════════════════════════════════

✅ تم رفع جميع التحديثات إلى GitHub بنجاح!

الآن يجب عليك تنفيذ الخطوات التالية يدوياً على Render:

───────────────────────────────────────────────────────────
📱 الخطوة 1: إصلاح Frontend
───────────────────────────────────────────────────────────

1. اذهب إلى: https://dashboard.render.com/
2. اختر خدمة Frontend (mimi-store-frontend)
3. اضغط على زر "Manual Deploy" في الأعلى
4. اختر "Clear build cache & deploy"
5. انتظر 5-10 دقائق حتى ينتهي البناء

───────────────────────────────────────────────────────────
🔧 الخطوة 2: إصلاح Backend
───────────────────────────────────────────────────────────

الطريقة الأسهل:
1. في Render Dashboard، اختر خدمة Backend
2. اضغط على "Manual Deploy" → "Clear build cache & deploy"
3. سيقوم build.sh بتشغيل migrations تلقائياً

أو الطريقة اليدوية (إذا لم تنجح الطريقة الأولى):
1. اختر خدمة Backend
2. اذهب إلى "Shell" من القائمة الجانبية
3. شغّل هذه الأوامر:
   cd backend
   python manage.py migrate
   python create_coupons.py

───────────────────────────────────────────────────────────
✅ الخطوة 3: التحقق من النجاح
───────────────────────────────────────────────────────────

Frontend:

   يجب أن يعمل الموقع بدون أخطاء

   يجب أن تظهر لوحة التحكم بدون أخطاء

───────────────────────────────────────────────────────────
❓ إذا واجهت مشاكل
───────────────────────────────────────────────────────────

Frontend لا يزال يعطي خطأ PostCSS:
→ تأكد من الضغط على "Clear build cache & deploy" وليس "Deploy"
→ تحقق من أن Branch = main في إعدادات الخدمة

Backend يعطي خطأ django_session:
→ شغّل migrations يدوياً من Shell
→ تأكد من اتصال قاعدة البيانات PostgreSQL
